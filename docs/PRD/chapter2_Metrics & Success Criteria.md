## **第二章：衡量指标与成功标准 (Metrics & Success Criteria)**

### **2.1 北极星指标 (North Star Metric)**

**W.A.N.U. (Weekly Awakened Node Users) - 周度节点唤醒用户数**

**定义：**
在过去7天内，至少成功"唤醒"一个智慧节点的用户总数。

**"唤醒"的定义：**
用户在复习环节中，将一个节点从"忘记/模糊"状态（首次复习或上次评分≤3）变为"掌握"状态（评分=5）。

**为何选择它：**

1. **价值导向**：不仅衡量活跃度，更衡量产品的核心价值（帮助用户真正记住知识）
2. **行为完整性**：覆盖了"输入 → AI处理 → 复习 → 掌握"的完整价值链
3. **增长驱动**：WANU的增长意味着用户获得了真实价值，会自然形成口碑传播
4. **可分解性**：可拆解为"创建节点数"和"复习参与度"两个子指标，便于定位问题

### **2.2 核心KPIs (Key Performance Indicators)**

#### **2.2.1 用户激活 (Activation)**

| 指标               | 定义                                     | 计算公式                        | V1.0目标 | 行业基准 |
| ------------------ | ---------------------------------------- | ------------------------------- | -------- | -------- |
| **新用户注册率**   | 打开小程序并完成微信授权的比例           | 授权用户数 / 小程序打开数       | ≥ 80%    | 70-85%   |
| **首次节点创建率** | 完成首次节点创建的新用户比例             | 创建节点的新用户 / 授权用户     | ≥ 50%    | 30-50%   |
| **核心价值体验率** | 体验到"思想共鸣"或完成首次复习的用户比例 | 体验核心功能用户 / 创建节点用户 | ≥ 40%    | -        |

**监控频率**：每日
**负责人**：产品经理

---

#### **2.2.2 用户活跃与参与 (Engagement)**

| 指标                           | 定义                             | V1.0目标   | 备注                 |
| ------------------------------ | -------------------------------- | ---------- | -------------------- |
| **DAU (日活跃用户)**           | 每日至少打开一次小程序的用户数   | 随自然增长 | 观察指标             |
| **WAU (周活跃用户)**           | 每周至少打开一次小程序的用户数   | 随自然增长 | 观察指标             |
| **MAU (月活跃用户)**           | 每月至少打开一次小程序的用户数   | 随自然增长 | 观察指标             |
| **DAU/MAU**                    | 衡量用户粘性                     | ≥ 20%      | 工具类产品通常15-25% |
| **核心用户节点创建频率 (P50)** | 活跃用户每周创建节点数的中位数   | ≥ 3个/周   | 核心指标             |
| **复习会话完成率**             | 完成当日所有复习的用户比例       | ≥ 30%      | 调整后目标           |
| **单次复习平均节点数**         | 用户单次复习环节平均复习的节点数 | ≥ 3个      | 观察指标             |

**监控频率**：每周
**负责人**：数据分析师

---

#### **2.2.3 用户留存 (Retention)**

| 指标                    | 定义                           | V1.0目标 | 行业基准 | 重要性 |
| ----------------------- | ------------------------------ | -------- | -------- | ------ |
| **次日留存率 (Day 1)**  | 注册后第1天再次打开的用户比例  | ≥ 40%    | 35-45%   | ⭐⭐⭐    |
| **3日留存率 (Day 3)**   | 注册后第3天再次打开的用户比例  | ≥ 25%    | 20-30%   | ⭐⭐⭐    |
| **7日留存率 (Day 7)**   | 注册后第7天再次打开的用户比例  | ≥ 18%    | 10-15%   | ⭐⭐⭐⭐   |
| **30日留存率 (Day 30)** | 注册后第30天再次打开的用户比例 | **≥ 8%** | 5-8%     | ⭐⭐⭐⭐⭐  |

**关键决策指标：**

- 如果30日留存率 < 5%，需立即启动用户访谈，重新评估产品核心价值
- 如果30日留存率 ≥ 10%，可考虑提前进入V1.1规划

**监控频率**：每周（Cohort分析）
**负责人**：产品经理 + 数据分析师

---

#### **2.2.4 AI质量与体验 (AI Quality)**

| 指标                 | 定义                             | V1.0目标 | 监控方式           |
| -------------------- | -------------------------------- | -------- | ------------------ |
| **AI内容好评率**     | 用户对AI生成内容点赞的比例       | ≥ 85%    | 节点详情页反馈数据 |
| **AI内容差评率**     | 用户对AI生成内容点踩的比例       | ≤ 10%    | 节点详情页反馈数据 |
| **内容二次编辑率**   | 用户对AI生成内容进行编辑的比例   | ≤ 25%    | 编辑行为埋点       |
| **"思想共鸣"有效率** | 用户对AI关联推荐点赞的比例       | ≥ 75%    | 调整后目标         |
| **AI处理失败率**     | AI处理异常/超时的比例            | ≤ 2%     | 服务端日志         |
| **ASR准确率**        | 语音转文本的准确率（抽样评估）   | ≥ 90%    | 人工抽样100条/周   |
| **OCR准确率**        | 图片文字识别的准确率（抽样评估） | ≥ 85%    | 人工抽样50条/周    |

**负责人**：算法工程师Lead + 产品经理

#### **2.2.5 成本效率 (Cost Efficiency)**

| 指标                   | 定义                         | V1.0目标 | 监控方式   |
| ---------------------- | ---------------------------- | -------- | ---------- |
| **单节点AI成本**       | 创建一个节点的平均AI服务成本 | ≤ ¥0.60  | 详见第九章 |

| **AI成本占比**         | AI成本占总运营成本的比例     | ≤ 60%    | 财务系统   |

**负责人**：技术Lead + 财务

### **2.3 指标监控体系**

**实时看板（每日更新）：**

- DAU/WAU/MAU 趋势图
- 新增用户与激活漏斗

**成本看板（每周更新）：**

- AI服务调用量与成本
- 各服务商成本占比
- 单用户成本趋势
