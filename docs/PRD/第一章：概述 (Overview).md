## 第一章：概述 (Overview)

### 1.1 项目背景与机遇

在当前信息爆炸的时代，个体面临着前所未有的"信息焦虑"与"知识内化"挑战。海量的信息快速流过，却难以沉淀为个人的智慧。现有的笔记和知识管理工具，大多停留在"被动式仓储"阶段，用户需要耗费大量精力进行手动的整理、归纳和复习，摩擦力巨大，难以坚持。

与此同时，以大语言模型（LLM）和多模态技术为代表的AI浪潮，为解决这一根本性痛点提供了历史性的机遇。AI原生应用不再是传统软件的辅助或增强，而是能够重塑整个信息处理与知识内化流程的全新物种。

"Weave (织念)"正是在这一背景下应运而生。我们预见到，未来每个知识工作者都将拥有一个与自己心意相通、共同进化的"第二心智"。Weave的目标，就是成为这个领域的定义者。

---

### 1.2 产品愿景与V1.0 (MVP) 目标

#### 产品愿景 (Vision)

**"将信息焦虑转化为智慧自信 (From Information Anxiety to Intellectual Confidence)"**

我们致力于打造一个"思想的编织者"与"智慧的共生体"，通过AI无感地捕捉、连接用户的思想碎片，并将其反哺、内化，帮助用户构建一个不断生长、永不遗忘的个人知识网络。

#### **V1.0 (MVP) 核心目标 (Goal)**

**以最小成本、最快速度，验证三个核心假设：**

| 核心假设                                     | 验证指标                     | 成功标准 |
| -------------------------------------------- | ---------------------------- | -------- |
| **H1: 用户愿意用"无摩擦"方式高频捕捉信息**   | 核心用户周均节点创建数 (P50) | ≥ 3个/周 |
| **H2: AI自动化处理能提供"魔法体验"**         | AI处理内容好评率             | ≥ 85%    |
| **H3: 用户会遵循AI推送的复习计划并形成习惯** | 30日留存率                   | ≥ 8%     |

**MVP原则：**

- ✅ **功能最小化**：只保留验证核心假设所需的功能
- ✅ **技术务实化**：优先选择成熟、低成本方案
- ✅ **体验完整化**：虽简单但必须形成完整的价值闭环

### **1.3 目标用户 (Target Audience)**

V1.0阶段，我们聚焦于对核心价值感知最敏锐的**种子用户**。

#### **核心用户画像: Alex (产品经理)**

**基本信息：**

- **年龄**：28岁
- **职业**：一线城市互联网产品经理
- **教育背景**：本科及以上
- **技术接受度**：高，是新产品的早期采纳者

**日常场景：**

- 每天需要处理大量信息（行业报告、竞品分析、用户反馈、会议纪要）
- 持续学习新知识（AI、设计思维、增长方法论）
- 通勤时间（地铁/开车）约2小时，习惯听播客、看公众号

**核心目标 (Goals):**

1. 高效内化新知识，并应用于工作场景
2. 建立自己的产品方法论知识体系
3. 在快节奏的工作中，不错过任何灵感火花
4. 在面试、述职、做方案时能系统性地调用知识

**痛点 (Frustrations):**

| 痛点                             | 现状                                    | 后果                    |
| -------------------------------- | --------------------------------------- | ----------------------- |
| **"收藏从未停止，学习从未开始"** | 微信收藏、Notion剪藏了300+篇文章        | 90%从未打开，产生焦虑感 |
| **"灵感转瞬即逝"**               | 会议/通勤想到好点子，用备忘录记录很繁琐 | 80%的灵感丢失           |
| **"知识是孤岛"**                 | 学到的知识点很零散，无法串联            | 面试时"嘴里有话倒不出"  |
| **"知道要复习，但就是懒"**       | 知道艾宾浩斯曲线，但手动制作Anki太痛苦  | 从未坚持超过2周         |

**Weave如何帮助Alex (Value Proposition):**

```
场景1：通勤路上听播客
传统方式：听完就忘，或打开备忘录手动记录（体验割裂）
Weave方式：长按麦克风说"刚才播客提到的AARRR模型..."，3秒后生成结构化节点

场景2：阅读行业报告
传统方式：截图保存到相册，事后很难找到，也不会整理
Weave方式：截图上传，AI自动提取关键信息并生成节点

场景3：知识体系构建
传统方式：需要手动整理思维导图，耗时且难坚持
Weave方式：AI自动发现"AARRR模型"与之前记录的"用户生命周期"的关联

场景4：知识复习
传统方式：依靠自律定期翻看笔记（很难做到）
Weave方式：每天早上收到3个即将遗忘的知识点推送，地铁上5分钟完成复习
```

### **1.4 V1.0 (MVP) 范围**

#### **IN SCOPE (范围内)**

**核心平台：**

- ✅ 微信小程序（单端）

**核心功能：**

- ✅ **US1**: 新用户引导与首次体验
- ✅ **US2**: 语音输入（批量处理版）
- ✅ **US3**: 截图/拍照输入
- ✅ **US4**: AI自动生成智慧节点
- ✅ **US5**: 思想共鸣（关键词匹配版）
- ✅ **US6**: 基于SM-2算法的记忆复习
- ✅ **US7**: 节点编辑与删除
- ✅ **US8**: 数据导出

**AI能力：**

- ✅ 语音转文本（ASR）
- ✅ 图片文字识别（OCR）
- ✅ 内容精炼与结构化（LLM）
- ✅ 简单关联发现（关键词+TF-IDF）

**人机协同：**

- ✅ AI处理过程可追溯
- ✅ 用户反馈机制（👍/👎）
- ✅ 内容二次编辑能力

---

#### **OUT OF SCOPE (范围外，规划至V1.1+)**

| 功能                     | 原因                                   | 规划版本 |
| ------------------------ | -------------------------------------- | -------- |
| **实时流式ASR**          | 技术复杂度高，批量处理已能验证核心假设 | V1.1     |
| **向量检索版"思想共鸣"** | 成本高，关键词匹配已能提供80%价值      | V1.1     |
| **可视化知识图谱**       | 开发周期长，非验证核心假设必需         | V1.2     |
| **URL链接输入**          | 需处理网页解析等复杂场景               | V1.1     |
| **全局搜索**             | 节点数量少时价值不明显                 | V1.1     |
| **个人成长报告**         | 需数据积累，首月用户无意义             | V1.3     |
| **多平台同步**           | 增加开发成本，MVP聚焦单端              | V2.0     |
| **社交功能**             | 与核心价值无关                         | 待定     |
| **付费订阅**             | 需先验证价值                           | V1.2     |

### **1.5 成功的定义 (Definition of Success)**

**V1.0 MVP被视为成功，当且仅当满足：**

**定量指标：**

- ✅ **30日留存率 ≥ 8%** （核心指标）
- ✅ **W.A.N.U. (周度节点唤醒用户数) ≥ 100** （北极星指标）
- ✅ **AI内容好评率 ≥ 85%**
- ✅ **核心用户周均创建节点数 (P50) ≥ 3**

**定性指标：**

- ✅ 至少50%的深度访谈用户表示"Weave帮我解决了知识管理的痛点"
- ✅ 至少30%的用户主动向他人推荐产品（NPS > 30）
- ✅ 无严重的数据安全或隐私合规事故

**财务指标：**

- ✅ 单活跃用户月度AI成本 ≤ ¥4.00
- ✅ MVP总预算（开发+运营+AI成本）≤ ¥50万
